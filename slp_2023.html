About SLP

Semester-Long Projects (SLP) is a great program to enrich coding skills in a group setting. Participants are formed into groups of similar interests. Each group is led by a mentor to build a unique coding project. SLP includes workshops like git, inclusive design, and other topics to aid the project's completion. At the end of the semester, the groups come together for sheCode's SLP Demo Day to showcase their project!


We congratulate every member and team lead on finishing SLP of 2022!

Thank you for your 10 weeks of teamwork and dedication! You have accomplished these past 10 weeks and should be proud of all the work you have done. We hope that SLP has encouraged you to explore more about Computer Science and inspires you to continue pursuing projects outside of class. About the Project



Video Demo

Github

sheCodes has Friday Coding Sessions and wants to share the problem solving videos in one place where other people can also share their solutions. Also, because sheCodes co-president, Min, doesn’t like sharing Google Drive Folders. Thus, sheLearns is a web application that allows sheCodes members to view and upload their own coding practice videos.

Roles/Responsibilities:
Lead and Members all got a chance to work on the front-end, back-end, and database.

Tools

  • Database Management System - PostgreSQL
  • SQL Client Software Application - DBeaver
  • Framework - Express
  • JavaScript Library - ReactJS
  • Server - Node.js
  • API Platform - Postman

Challenges

  • Difficulty with creating PostgreSQL tables and didn’t know some psql keywords.
  • Difficulty with flex grids on the front-end; sometimes, the content inside a box would spill out and had to figure out how to properly use flexbox
  • Figuring out why the one post works and the other post returns not found.
  • Didn’t fully understand Express so didn’t pass different parameters.
  • Debugging why the postgresql couldn’t recognize the column name
  • Postgresql default to lower case. Uppercase parameters would be recognized as lowercase: By changing the column names to lowercase, it was solved.
  • Uploading videos
  • Since Video unstructured data, meaning they don’t have a fixed and organized data structure (doesn’t use rows and columns) it can not be stored into PostgreSQL
  • Debugging and making sure everyone has all the applications needed installed
  • Working with Windows and MacOs

What Was Learned

  • Learned how to create sql tables and have a better understanding of sql syntax
  • Takeaway: Plan to practice and study sql more for future projects that require a database
  • Learned how to use flexbox and improved css skills
  • Learned how to use REST APIs to POST, data processing and error handling
  • Learned the interaction between server and client
  • Got to practice REST APIs more
  • Learned to connect databases to the server-side

Given More Time...

  • Create and separate between production and development code
    • - Add more features:
  • Allow users to upload videos from their local drive (e.g. MP3/4 files)
  • Allow user to upload images and word documents
  • Add stronger authentication when login
  • Add like and save button
  • Show user upload history
  • Add chat box
